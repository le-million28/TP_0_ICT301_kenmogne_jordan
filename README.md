## Les principes SOLID

### 1. SRP – Single Responsibility Principle
Une classe ne doit avoir qu’une seule responsabilité  
Cela permet de faciliter la maintenance, la lisibilité et l’évolution du code.

---

### 2. OCP – Open/Closed Principle
Les classes doivent être ouvertes à l’extension mais fermées à la modification.  
On évite ainsi de casser le code existant lorsqu’on ajoute de nouvelles fonctionnalités.

---

### 3. LSP – Liskov Substitution Principle
Une classe fille doit pouvoir remplacer sa classe mère sans altérer le bon fonctionnement du programme.

---

### 4. ISP – Interface Segregation Principle
Il vaut mieux plusieurs interfaces spécifiques qu’une seule interface générale trop lourde.

---

### 5. DIP – Dependency Inversion Principle
Les modules de haut niveau ne doivent pas dépendre des modules de bas niveau,  
mais d’abstractions (interfaces).
